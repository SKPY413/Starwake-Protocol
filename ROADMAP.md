# STARWAKE PROTOCOL — ROADMAP

This roadmap uses named milestones instead of relying on numerical version nomenclature. A build number may still identify a specific archive, but progress is judged by milestone readiness.

Items may move as the design develops. Checked boxes require playtested evidence, not merely implemented code.


###############################################################################
PRE-BETA — COMPLETE THE GAME'S CORE IDENTITY
###############################################################################

Goal: All essential systems exist, function together, and communicate the intended Starwake experience.

- [ ] Define the complete core enemy-role roster
- [ ] Give every enemy a distinct tactical purpose and readable counterplay
- [ ] Complete the foundational boss framework and boss encounter language
- [ ] Complete the core upgrade and research structure
- [ ] Establish a stable economy across all difficulty settings
- [ ] Preserve a genuinely welcoming Easy mode
- [ ] Make Impossible demand pattern mastery rather than health inflation alone
- [ ] Stabilize desktop, mobile, and controller input
- [ ] Stabilize save and settings behavior
- [ ] Complete the essential audio mixer and voice-pressure safeguards
- [ ] Give anomalous enemies distinctive localized audio without arbitrarily replacing the soundtrack
- [ ] Standardize HUD, menus, feedback, and accessibility foundations
- [ ] Establish `ADMIN.md`, `TEST_REPORT.md`, `KNOWN_ISSUES.md`, and contributor SOP workflow
- [ ] Establish the canonical Lore Bible and plot framework
- [ ] Remove critical and repeatable run-ending defects

Pre-Beta exit question:

> Does Starwake already feel like itself, even before final content and polish?


###############################################################################
BETA — PROVE, BALANCE, AND POLISH THE COMPLETE EXPERIENCE
###############################################################################

Goal: The intended release experience is feature-complete enough for broad testing. Work shifts from invention toward validation, balance, usability, and reliability.

- [ ] Freeze the essential release feature set
- [ ] Conduct repeated external playtests using the shared rating system
- [ ] Track feedback patterns instead of reacting to isolated opinions
- [ ] Balance progression, rewards, upgrades, enemies, bosses, and wave pacing
- [ ] Complete onboarding, help, controls, and accessibility information
- [ ] Complete settings and persistence behavior
- [ ] Add player statistics and run summaries where they improve decisions
- [ ] Add achievements or records only when they reinforce existing play
- [ ] Establish story delivery that remains optional during active play
- [ ] Complete enemy, faction, boss, and location naming consistency
- [ ] Complete credits and attribution structure
- [ ] Test long runs and late-wave performance
- [ ] Test current desktop and mobile browsers
- [ ] Resolve all Critical issues and the release-blocking Major issues
- [ ] Document remaining known limitations honestly

Beta exit question:

> Can unfamiliar players understand, enjoy, and reliably complete the intended game loop without developer assistance?


###############################################################################
RELEASE — SHIP A COHERENT, STABLE, HONEST FIRST EDITION
###############################################################################

Goal: Publish a stable edition that represents the collaboration well and can be preserved indefinitely.

- [ ] Final gameplay and economy balance pass
- [ ] Final performance and compatibility pass
- [ ] Final interface and accessibility pass
- [ ] Final audio and soundtrack pass
- [ ] Final narrative continuity and terminology pass
- [ ] Final save compatibility and migration decision
- [ ] Final credits, licenses, acknowledgements, and collaboration statement
- [ ] Preserve the original human–AI collaborative project identity
- [ ] Create release notes and a clean distributable archive
- [ ] Preserve a verified release build separately from active development
- [ ] Publish known issues and support expectations
- [ ] Create store, website, or distribution materials only from verified release content
- [ ] Confirm that the release build can be launched from a clean extraction

Release principle:

> The first release does not need every possible idea. It must deliver the ideas it contains with purpose and care.


###############################################################################
POST-RELEASE UPDATES — IMPROVE WITHOUT DILUTING
###############################################################################

Potential update families—not commitments:

- [ ] Quality-of-life and accessibility updates
- [ ] New tactical enemy roles
- [ ] New boss encounters tied to established factions and mechanics
- [ ] New weapons or research paths with meaningful tradeoffs
- [ ] New locations or modes that justify their own audio and visual identity
- [ ] Narrative discoveries that reinterpret existing mechanics
- [ ] Community-requested improvements supported by repeated evidence
- [ ] Preservation updates for future browsers and platforms

Every post-release proposal must answer:

1. What player need does this address?
2. Why does it belong in Starwake?
3. What existing system does it reinforce?
4. What complexity and maintenance burden does it add?


###############################################################################
TEN-YEAR ANNIVERSARY — PRESERVE THE JOURNEY
###############################################################################

An optimistic long-range list for 2036. This is a time capsule, not a production commitment.

- [ ] Preserve and make the original 2026 build playable
- [ ] Add a prototype gallery or museum mode
- [ ] Include selected historical screenshots, notes, and discarded concepts
- [ ] Add creator commentary focused on actual decisions and lessons
- [ ] Remaster or respectfully restore signature music and sounds
- [ ] Add an anniversary cosmetic that does not affect balance
- [ ] Thank long-term players, testers, contributors, and preservation volunteers
- [ ] Publish a development timeline assembled from archived builds and documentation
- [ ] Preserve the statement: “A Human–AI Collaborative Experiment in Emergent Game Design.”
- [ ] Let the project history speak for itself without pretending the road was cleaner than it was

Anniversary question:

> Can a new player understand what Starwake became while an early player can still recognize where it began?
