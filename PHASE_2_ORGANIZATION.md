# Pre-Beta Engine Cleanup — Phase 2

Phase 2 organizes the monolithic runtime into maintainable source sections without changing browser behavior.

## Runtime strategy

`index.html` continues to load `game.js`. This avoids changing JavaScript scope, declaration order, or startup semantics. The editable source now lives under `src/`, and `game.js` is generated by concatenating those files in the exact order recorded in `src/manifest.json`.

This is deliberately safer than converting a mature 9,000-line IIFE directly to ES modules in one pass. Later cleanup phases can migrate individual systems behind explicit interfaces.

## Workflow

1. Edit the appropriate file under `src/`.
2. Run `python tools/build_game.py`.
3. Run `python tools/verify_build.py`.
4. Run `node --check game.js`.
5. Commit both the edited source file and regenerated `game.js`.

## Source map

- `00_bootstrap.js` — wrapper, storage guards, DOM, constants, data tables.
- `01_audio.js` — music and sound systems.
- `02_input_relics_coordinates.js` — input, relic metadata, damage types, camera and coordinates.
- `03_state_saves_debug.js` — state, run lifecycle, undo state, debug tools.
- `04_upgrades_player_relic_runtime.js` — upgrades, player weapons/movement, relic runtime.
- `05_enemies.js` — roster, generations, mutations, AI, carriers and bosses.
- `06_combat_pickups_effects.js` — collisions, explosions, pickups and transient effects.
- `07_background_rendering.js` — background and rendering.
- `08_ui_loop_input.js` — UI, main loop, touch and gamepad.
- `09_events_persistence_startup.js` — event binding, persistence and startup.

## Deployment

Vercel still needs only the normal static files. The `src/` and `tools/` directories are development sources; keeping them in GitHub makes future edits reviewable. `game.js` remains deployable without any server-side build step.
